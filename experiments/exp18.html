Experiment 16: Conjugate Gradient Method

1) Aim
Implement the Conjugate Gradient (CG) method to solve the linear system: $$Ax=b$$ with interpreted input: $$A=\begin{bmatrix}4 & 1\\1 & 3\end{bmatrix}, \qquad b=\begin{bmatrix}1\\2\end{bmatrix}$$
2) Description

Conjugate Gradient Method solves linear systems $Ax=b$ iteratively without forming matrix inverses—a computational miracle that works for large systems. The method builds a sequence of orthogonal (conjugate with respect to $A$) search directions, ensuring convergence in at most $n$ steps for an $n\times n$ matrix. In practice, with preconditioning, it converges much faster and has become the standard for large sparse systems in scientific computing.

The key insight is that solving $Ax=b$ (when $A$ is symmetric positive definite) is equivalent to minimizing the quadratic form $\tfrac{1}{2}x^TAx - b^Tx$. Conjugate Gradient cleverly exploits this: each step moves along a direction conjugate to previous directions with respect to $A$, ensuring optimal progress. The algorithm only requires matrix-vector products, not matrix storage, making it memory-efficient for large-scale problems.

CG's efficiency and scalability made it revolutionary in scientific computing. It extends to non-symmetric and non-square systems (GMRES, BiCG), and preconditioned variants solve ill-conditioned systems efficiently. Understanding CG principles helps grasp modern optimization for large-scale machine learning.

3) Important Formulas

System: $$Ax=b$$

Initialize: $$x_0=0, \quad r_0=b-Ax_0, \quad p_0=r_0$$

Step size: $$\alpha_k=\frac{r_k^Tr_k}{p_k^TAp_k}$$

Solution update: $$x_{k+1}=x_k+\alpha_k p_k$$

Residual update: $$r_{k+1}=r_k-\alpha_kAp_k$$

Direction coefficient: $$\beta_k=\frac{r_{k+1}^Tr_{k+1}}{r_k^Tr_k}$$

Direction update: $$p_{k+1}=r_{k+1}+\beta_k p_k$$

Stopping condition: $$\|r_k\|_2 < \varepsilon$$

4) Important Functions Used
  • conjugate_gradient(A, b, x0, tol, max_iter): implements CG iterations.
  • np.dot(...): inner products for $\alpha$ and $\beta$.
  • np.linalg.norm(...): residual norm for stopping.
5) Algorithm (Step by Step)
  1. Choose initial guess $x_0$.
  2. Compute residual $r_0=b-Ax_0$.
  3. Set search direction $p_0=r_0$.
  4. Compute $\alpha_k$.
  5. Update $x_{k+1}$ and $r_{k+1}$.
  6. If residual norm is small, stop.
  7. Compute $\beta_k$.
  8. Update direction $p_{k+1}$.
  9. Repeat until convergence.
6) Mathematical Procedure (First Iteration)

Take $x_0=[0,0]^T$. Then: $r_0=b-Ax_0=b=\begin{bmatrix}1\\2\end{bmatrix}, \quad p_0=r_0$

Compute: $$Ap_0=\begin{bmatrix}4 & 1\\1 & 3\end{bmatrix} \begin{bmatrix}1\\2\end{bmatrix} = \begin{bmatrix}6\\7\end{bmatrix}$$
Step size: $$\alpha_0=\frac{r_0^Tr_0}{p_0^TAp_0} = \frac{1^2+2^2}{[1,2]\cdot[6,7]} = \frac{5}{20}=0.25$$
Update: $$x_1=x_0+0.25p_0 = \begin{bmatrix}0.25\\0.5\end{bmatrix}$$ $$r_1=r_0-0.25Ap_0 = \begin{bmatrix}1\\2\end{bmatrix}-0.25 \begin{bmatrix}6\\7\end{bmatrix} = \begin{bmatrix}-0.5\\0.25\end{bmatrix}$$

Continue similarly until residual is sufficiently small.

7) Given Input
  • $A=\begin{bmatrix}4&1\\1&3\end{bmatrix}$
  • $b=\begin{bmatrix}1\\2\end{bmatrix}$
  • Initial guess: $x_0=[0,0]^T$
  • Tolerance: $10^{-10}$
8) Python Implementation & Output
import numpy as np # Given A = np.array([[4.0, 1.0], [1.0, 3.0]]) b = np.array([1.0, 2.0]) # Initial guess x = np.array([0.0, 0.0]) # Initial residual and direction r = b - A @ x p = r.copy() for _ in range(10): # enough for 2x2 (max 2 steps theoretically) Ap = A @ p alpha = (r @ r) / (p @ Ap) x = x + alpha * p r_new = r - alpha * Ap # Stop if converged if np.linalg.norm(r_new) < 1e-10: break beta = (r_new @ r_new) / (r @ r) p = r_new + beta * p r = r_new # Output print("function: Ax = b") print("A = ", A) print("b =", b) print("Solution x =", np.round(x, 6)) print("Check A@x =", np.round(A @ x, 6)) print("Residual =", np.round(b - A @ x, 10))
