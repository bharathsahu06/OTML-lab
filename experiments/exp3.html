Experiment 3: Stochastic Gradient Descent for f(x) = x^2

📌 Aim

To implement Stochastic Gradient Descent (SGD) for the function f(x) = x^2 using the sample values x = [1, 2, 3, 4] and y = [1, 4, 9, 16]. This experiment highlights how stochastic-style updates can still move the variable toward the minimum even when each step is noisy.

📝 Description

Stochastic Gradient Descent (SGD) is a variation of gradient descent that updates the parameters using only a small portion of the available information at each step. This makes each update inexpensive and allows the method to react quickly while optimization is in progress.

Because the updates are stochastic, the optimization path is noisier than standard gradient descent, but the method scales well and is central to modern machine learning. Its behavior depends on the learning rate, data ordering, and the tradeoff between speed and stability.

  • Updates happen more frequently than in full batch gradient descent
  • Each step is cheap to compute but less stable than a full gradient update
  • Noisy updates can still move the method toward a good solution over time
🔧 Algorithm
Step 1: Start with an initial value such as x = 5.0
Step 2: Choose a learning rate and number of epochs
Step 3: Estimate a stochastic gradient using 2x plus a small random variation
Step 4: Update x using the stochastic gradient
Step 5: Repeat the process until x moves close to the minimum
📐 Formulas
Objective Function:
f(x) = x2
Given Data:
x = [1, 2, 3, 4], y = [1, 4, 9, 16]
True Gradient:
df/dx = 2x
Stochastic Sample Gradient:
g_i = 2x_i for randomly selected sample x_i
SGD Update Rule:
x_i ← x_i - αg_i = x_i - 2αx_i
📊 Mathematical Procedure

Given sample values: x = [1, 2, 3, 4] and y = [1, 4, 9, 16]. These values satisfy y = x^2.

Use SGD over randomly selected sample points x_i with α = 0.1.

Sample update example:
pick x_i = 4
g_i = 2x_i = 8
updated x_i = 4 - 0.1(8) = 3.2
Another sample update:
pick x_i = 2
g_i = 4
updated x_i = 2 - 0.1(4) = 1.6

Repeating random sample updates across epochs reduces every x_i toward 0, so y_i = x_i^2 also moves toward 0.

Interpretation: this is stochastic because one sample is updated at a time, but the same objective f(x)=x^2 is minimized.

💻 Simple Python Program
import random random.seed(42) # SGD for f(x)=x^2 using given sample values x_data = [1, 2, 3, 4] y_data = [1, 4, 9, 16] print(f"Given x values: {x_data}") print(f"Given y values: {y_data}") learning_rate = 0.1 epochs = 20 for epoch in range(epochs): indices = [0, 1, 2, 3] random.shuffle(indices) for idx in indices: grad = 2 * x_data[idx] x_data[idx] = x_data[idx] - learning_rate * grad y_data[idx] = x_data[idx] ** 2 if epoch % 5 == 0: avg_loss = sum(y_data) / len(y_data) print(f"Epoch {epoch}: x = {[round(v,4) for v in x_data]}, Avg Loss = {avg_loss:.6f}") print(f"\nFinal x values = {[round(v,6) for v in x_data]}") print(f"Final y values = {[round(v,6) for v in y_data]}")
