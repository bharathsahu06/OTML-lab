Experiment 3: Stochastic Gradient Descent (SGD)

1) Aim
Implementation of Stochastic Gradient Descent to learn $w$ in $f(x)=w x^2$ for the given data:
  • Input $x=[1,2,3,4]$
  • Target $y=[1,4,9,16]$
2) Description

Stochastic Gradient Descent (SGD) is a variant of gradient descent that updates parameters using single samples (or small random subsets) rather than the entire dataset. This introduces noise into gradient estimates but provides critical practical advantages: lower memory requirements, ability to process streaming data, and often faster convergence in practice due to the noise acting as a regularizer.

The key insight is the trade-off between gradient accuracy and computational speed. Full-batch gradient descent provides accurate gradients but is slow on large datasets. SGD trades some accuracy for speed and generalization. The noise from individual sample gradients prevents overfitting and helps escape shallow local minima. This makes SGD particularly valuable for large-scale machine learning where data volume makes batch processing infeasible.

SGD's effectiveness depends on careful learning rate scheduling. A fixed learning rate may cause oscillations around the optimum, so adaptive schedules (like learning rate decay) are often necessary. Despite being noisier, SGD often generalizes better than full-batch methods due to the implicit regularization from gradient noise.

3) Important Formulas

Model: $$\hat{y}_i=w x_i^2$$

Sample loss (squared error): $$L_i=(\hat{y}_i-y_i)^2=(w x_i^2-y_i)^2$$

Sample gradient: $$\frac{dL_i}{dw}=2(w x_i^2-y_i)x_i^2$$

SGD update: $$w\leftarrow w-\alpha\frac{dL_i}{dw}$$

Final MSE (for reporting): $$J(w)=\frac{1}{n}\sum_{i=1}^{n}(w x_i^2-y_i)^2$$

4) Important Functions Used in Program
  • np.random.permutation(n): shuffles sample order each epoch.
  • np.mean(...): computes Mean Squared Error over the full dataset.
5) Algorithm (Step by Step)
  1. Read an initial value for $w$ and set hyperparameters ($\alpha$, epochs).
  2. For each epoch, shuffle the sample order.
  3. For each sample $(x_i,y_i)$, compute $\hat{y}_i=w x_i^2$ and update $w$ using the sample gradient.
  4. After the epoch, compute and print the full-dataset MSE.
  5. After all epochs, report final $w$ and final loss.
6) Mathematical Procedure (First Two SGD Updates)

Given: $x=[1,2,3,4]$, $y=[1,4,9,16]$, model $\hat{y}=w x^2$.

For first sample $(x_1,y_1)=(1,1)$: $$\hat{y}_1=w\cdot 1^2=w, \quad e_1=w-1$$ $$\frac{dL_1}{dw}=2(w-1)\cdot 1^2=2(w-1),\quad w\leftarrow w-\alpha\,2(w-1)$$
For second sample $(x_2,y_2)=(2,4)$: $$\hat{y}_2=w\cdot 2^2=4w, \quad e_2=4w-4$$ $$\frac{dL_2}{dw}=2(4w-4)\cdot 2^2=8(4w-4),\quad w\leftarrow w-\alpha\,8(4w-4)$$

SGD continues similarly for the remaining samples and epochs.

7) Given Input
  • $x=[1,2,3,4]$
  • $y=[1,4,9,16]$
  • Initial $w$ (user input)
  • Learning rate $\alpha=0.001$
  • Epochs $=20$
8) Python Implementation & Output
import numpy as np # Data x = np.array([1, 2, 3, 4], dtype=float) y = np.array([1, 4, 9, 16], dtype=float) # Input w = float(input("Enter initial w: ")) # Hyperparameters lr = 0.001 epochs = 20 print("\nEpoch\t w\t\t Loss") for epoch in range(1, epochs + 1): # Shuffle (important in SGD) indices = np.random.permutation(len(x)) x_shuffled = x[indices] y_shuffled = y[indices] # SGD updates (one by one) for xi, yi in zip(x_shuffled, y_shuffled): y_pred = w * xi**2 error = y_pred - yi grad = 2 * error * xi**2 w = w - lr * grad # Loss after full pass y_pred_all = w * x**2 loss = np.mean((y_pred_all - y)**2) print(f"{epoch}\t {w:.6f}\t {loss:.6f}") # Result print("f(x) = w * x^2") print("\nOptimal w =", round(w, 6)) print("Final Loss =", round(loss, 6))
