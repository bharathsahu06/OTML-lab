Experiment 3: Stochastic Gradient Descent (SGD)

📌 Aim

To implement Stochastic Gradient Descent (SGD) for minimizing f(x) = x². Unlike standard gradient descent that uses all data at once, SGD updates using one random sample at a time, making it faster for large datasets.

📝 Description

Stochastic Gradient Descent (SGD) is a variation of gradient descent in which the model is updated using one sample at a time instead of the full dataset. This makes each update very cheap and allows the algorithm to react quickly to data.

Because every step is based on limited information, the optimization path becomes noisier than standard gradient descent. That noise is often acceptable in practice because SGD scales well to large datasets and still moves toward the minimum on average over many updates.

  • Updates happen more frequently than in batch gradient descent
  • Each step is computationally cheap but less stable
  • The method is especially useful for large-scale and online learning problems
🔧 Algorithm
Step 1: Start with random x
Step 2: For each epoch (pass through data):
Step 3: Randomly shuffle data
Step 4: For each sample, compute gradient and update x
Step 5: Repeat until convergence
📐 Formulas
Objective Function:
f(x) = x²
Gradient:
∂f/∂x = 2x
SGD Update (per sample):
x(n+1) = x(n) - α × ∇f(x(n), sample)
📊 Mathematical Procedure

SGD works similarly to gradient descent, but processes one random sample at a time:

Initialization: x = 5.0, α = 0.01
Random update 1:
Gradient = 2 × 5 = 10
x = 5 - 0.01 × 10 = 4.9
Random update 2:
Gradient = 2 × 4.9 = 9.8
x = 4.9 - 0.01 × 9.8 = 4.81
💻 Simple Python Program
