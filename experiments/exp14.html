Experiment 14: Minibatch Gradient Descent for Linear Regression

1) Aim
Minimize the loss function of linear regression using minibatch gradient descent for:
  • $x=[1,2,3,4]$
  • $y=[5,7,9,11]$
2) Description

Minibatch Gradient Descent provides a practical middle ground between pure stochastic and full-batch gradient descent. By processing small groups of samples together, minibatch methods reduce gradient variance compared to SGD while remaining faster than full-batch methods on large datasets. This is the standard approach in modern deep learning, where hardware (GPUs) is designed to process batches efficiently.

The key advantage is computational efficiency combined with improved generalization. Processing minibatches amortizes overhead (data loading, parallelization) across multiple samples. The batch size creates a hyperparameter controlling the variance-efficiency trade-off: larger batches give more accurate gradients but require more compute; smaller batches provide more frequent updates with noisier gradient estimates. A moderate batch size (typically 32-256) balances these concerns.

Minibatch gradient descent naturally incorporates shuffling, which provides computational benefits and acts as a regularizer. The iteration count per epoch (number of batches) creates implicit learning rate scheduling: completing many small updates per epoch leads to different learning dynamics than fewer larger updates per epoch.

3) Important Formulas

Model: $$\hat{y}_i = wx_i + b$$

Minibatch MSE (batch size $m$): $$J_B(w,b)=\frac{1}{m}\sum_{i\in B}(\hat{y}_i-y_i)^2$$

Gradients for minibatch $B$: $$\frac{\partial J_B}{\partial w}=\frac{2}{m}\sum_{i\in B}(\hat{y}_i-y_i)x_i, \qquad \frac{\partial J_B}{\partial b}=\frac{2}{m}\sum_{i\in B}(\hat{y}_i-y_i)$$

Update rules: $$w\leftarrow w-\alpha\frac{\partial J_B}{\partial w}, \qquad b\leftarrow b-\alpha\frac{\partial J_B}{\partial b}$$

Final full-data MSE: $$J(w,b)=\frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i-y_i)^2$$

4) Important Functions Used
  • minibatch_gradient_descent(...): performs minibatch updates.
  • mse_loss(...): computes full-dataset MSE.
  • np.random.default_rng(...).shuffle(...): shuffles sample order each epoch.
5) Algorithm (Step by Step)
  1. Initialize parameters $w=0$, $b=0$.
  2. Set learning rate, epochs, and batch size.
  3. Shuffle indices at each epoch.
  4. Create minibatches from shuffled data.
  5. For each minibatch, compute predictions and errors.
  6. Compute gradients and update $w,b$.
  7. Repeat for all minibatches and epochs.
  8. Compute final loss and predictions.
6) Mathematical Procedure (Every Step Shown)

Given: $(x,y)=\{(1,5),(2,7),(3,9),(4,11)\},\quad \alpha=0.01,\quad m=2$

Initialize: $w^{(0)}=0,\quad b^{(0)}=0$

For clear manual calculation, take first epoch minibatches as: Batch 1: $(1,5),(2,7)$, Batch 2: $(3,9),(4,11)$

Epoch 1, Batch 1

1. Predictions: $$\hat{y}_1=0(1)+0=0,\qquad \hat{y}_2=0(2)+0=0$$
2. Errors: $$e_1=0-5=-5,\qquad e_2=0-7=-7$$
3. Gradient w.r.t. $w$: $$\frac{\partial J_{B1}}{\partial w}=\frac{2}{2}[(-5)(1)+(-7)(2)] = -5-14=-19$$
4. Gradient w.r.t. $b$: $$\frac{\partial J_{B1}}{\partial b}=\frac{2}{2}[(-5)+(-7)] = -12$$
5. Update: $$w^{(1)}=0-0.01(-19)=0.19,\qquad b^{(1)}=0-0.01(-12)=0.12$$

Epoch 1, Batch 2

Use updated parameters $w=0.19, b=0.12$.

1. Predictions: $$\hat{y}_3=0.19(3)+0.12=0.69,\qquad \hat{y}_4=0.19(4)+0.12=0.88$$
2. Errors: $$e_3=0.69-9=-8.31,\qquad e_4=0.88-11=-10.12$$
3. Gradient w.r.t. $w$: $$\frac{\partial J_{B2}}{\partial w}=\frac{2}{2}[(-8.31)(3)+(-10.12)(4)] = -24.93-40.48=-65.41$$
4. Gradient w.r.t. $b$: $$\frac{\partial J_{B2}}{\partial b}=\frac{2}{2}[(-8.31)+(-10.12)] = -18.43$$
5. Update: $$w^{(2)}=0.19-0.01(-65.41)=0.8441,\qquad b^{(2)}=0.12-0.01(-18.43)=0.3043$$

End of Epoch 1 Loss Check (full data)

Using $w=0.8441, b=0.3043$: $$\hat{y}=[1.1484,\;1.9925,\;2.8366,\;3.6807]$$

Errors: $$\hat{y}-y=[-3.8516,\;-5.0075,\;-6.1634,\;-7.3193]$$

MSE: $$J=\frac{1}{4}\left[(-3.8516)^2+(-5.0075)^2+(-6.1634)^2+(-7.3193)^2\right] \approx 32.87$$

The same sequence is repeated for all epochs until final convergence near $\hat{y}\approx [5,7,9,11]$.

7) Given Input
  • $x=[1,2,3,4]$
  • $y=[5,7,9,11]$
  • Learning rate $\alpha=0.01$
  • Epochs $=1000$
  • Minibatch size $=2$
