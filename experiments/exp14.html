Experiment 14: Mini-batch Gradient Descent (Dataset 2)

📌 Aim

Similar to Experiment 12 but with different data: x=[1,2,3,4], y=[5,7,9,11]. Demonstrates that mini-batch GD works consistently across different datasets.

📝 Description

Mini-batch gradient descent is a compromise between batch gradient descent and stochastic gradient descent. Instead of using the full dataset or a single sample for each update, it computes gradients on small groups of samples called mini-batches.

This gives a useful balance between computational efficiency and stability. The method is widely used in machine learning because batch size can be tuned to control update noise, memory use, speed, and convergence behavior during training.

  • Each mini-batch provides a partial but useful estimate of the full gradient
  • Repeated epochs gradually improve the model parameters
  • Batch size affects computational cost, stability, and convergence behavior
🔧 Algorithm
Step 1: Initialize weights w and bias b
Step 2: Create mini-batches from data
Step 3: For each batch, compute MSE loss
Step 4: Calculate gradients
Step 5: Update parameters
💻 Simple Python Program
import numpy as np X = np.array([1, 2, 3, 4]) y = np.array([5, 7, 9, 11]) w = 0.0 b = 0.0 lr = 0.01 batch_size = 2 epochs = 50 for epoch in range(epochs): indices = np.random.permutation(len(X)) for i in range(0, len(X), batch_size): batch_indices = indices[i:i+batch_size] X_batch = X[batch_indices] y_batch = y[batch_indices] y_pred = w*X_batch + b dw = -2/batch_size * np.sum((y_batch - y_pred) * X_batch) db = -2/batch_size * np.sum(y_batch - y_pred) w -= lr * dw b -= lr * db y_pred_all = w*X + b loss = np.mean((y - y_pred_all)**2) if epoch % 10 == 0: print(f"Epoch {epoch}: Loss={loss:.4f}, w={w:.4f}, b={b:.4f}") print(f"\nFinal: y = {w:.4f}x + {b:.4f}")
