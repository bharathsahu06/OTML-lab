Experiment 15: Distributed System Staleness

Home / Experiment 15
Aim

To analyze the convergence properties and stability of distributed optimization in the presence of communication delays (staleness) between workers and the parameter server.

Description

In large-scale asynchronous distributed systems, workers can be "stragglers" (slow nodes). When a worker submits a gradient computed on an old version of the parameters, it is called a "stale gradient". This experiment models such staleness and its effect on the descent trajectory.

Stale Update: \( \mathbf{w}_{t+1} = \mathbf{w}_t - \eta \cdot \nabla L(\mathbf{w}_{t-\tau}) \)

Where \( \tau \) is the staleness factor (number of steps lag).

Algorithm
  1. Initialize global parameters and a history log of parameter states.
  2. For each iteration:
    • Simulate an asynchronous worker by selecting a "stale" version of parameters from history.
    • Compute the gradient based on the stale parameters.
    • Update the current parameters using this stale gradient.
    • Observe the oscillation or deviation from the true path of convergence.
Mathematical Analysis

As staleness \( \tau \) increases, the model may diverge if the learning rate \( \eta \) is not reduced. The effective gradient becomes increasingly inaccurate as the current position \( \mathbf{w}_t \) moves away from the anchor point \( \mathbf{w}_{t-\tau} \).

Simple Python Program
import numpy as np

def f(x): return x**2
def grad_f(x): return 2*x

x_history = [10.0]
x_current = 10.0
lr = 0.1
staleness = 3 # Lag of 3 iterations

for t in range(20):
    # Determine the stale index (t - tau)
    stale_idx = max(0, len(x_history) - 1 - staleness)
    x_stale = x_history[stale_idx]
    
    # Compute gradient on STALE x
    grad = grad_f(x_stale)
    
    # Update CURRENT x
    x_current = x_current - lr * grad
    x_history.append(x_current)
    
    print(f"t={t:2} | x_current={x_current:7.4f} | grad(x_stale)={grad:7.4f}")

print(f"Final Value with staleness {staleness}: {x_current}")
Expected Output
Output showing oscillation/lag:
t= 0 | x_current= 8.0000 | grad(x_stale)= 20.0000
t= 1 | x_current= 6.0000 | grad(x_stale)= 20.0000
...
Final result will converge more slowly than Batch GD (tau=0).
