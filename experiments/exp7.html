Experiment 7: LP with and without PuLP

Home / Experiment 7
Aim

To implement linear programming for the objective \( z = 3x + 2y \) subject to \( x + y \ge 4, x \ge 0, y \ge 0 \) using both the PuLP library and manual corner-point enumeration.

Description

Linear Programming (LP) problems involve optimizing a linear objective under linear inequality constraints. PuLP is a high-level library that interfaces with solvers like CBC, while the manual method involves finding the vertices of the feasible region and checking the objective values at each point.

Problem: \( \min 3x + 2y \)
Subject to: \( x + y \ge 4, x \ge 0, y \ge 0 \)
Algorithm
  1. Part A (Without PuLP):
    • Identify feasible region corner points: (0, 4) and (4, 0).
    • Compute objective \( z = 3x + 2y \) for all corners.
    • Select the corner with the minimum value.
  2. Part B (With PuLP):
    • Define decision variables \( x, y \ge 0 \).
    • Formulate the objective function and the constraints.
    • Solve using \( \texttt{prob.solve()} \).
Simple Python Program
from pulp import *

# 1. Without PuLP (Manual Enumeration)
corners = [(0, 4), (4, 0)]
print("Manual Corner Point Results:")
for x, y in corners:
    print(f"({x}, {y}) | z = {3*x + 2*y}")

# 2. With PuLP
prob = LpProblem("Simple_LP", LpMinimize)
x = LpVariable("x", lowBound=0)
y = LpVariable("y", lowBound=0)
prob += 3*x + 2*y
prob += x + y >= 4
prob.solve(PULP_CBC_CMD(msg=0))

print(f"\nPuLP Result: x={value(x)}, y={value(y)}, Objective={value(prob.objective)}")
Expected Output
Manual Corner Point Results:
(0, 4) | z = 8
(4, 0) | z = 12

PuLP Result: x=0.0, y=4.0, Objective=8.0
(Solution: Point (0,4) is optimal).
