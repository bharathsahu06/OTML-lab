Experiment 09: Mini-batch Gradient Descent

Home / Experiment 09
Aim

To implement Mini-batch Gradient Descent for linear regression, a compromise between Batch GD and Stochastic GD that uses small subsets of data to update parameters.

Description

Mini-batch Gradient Descent divides the training dataset into small batches and performs an update for each batch. This stabilizes the update process compared to SGD and is more computationally efficient than Batch GD as it leverages vectorized operations.

Dataset: \( X = [1, 2, 3, 4], Y = [3, 5, 7, 9] \).
Batch Size: 2.

Algorithm
  1. Initialize parameters \( w \) and \( b \).
  2. For each epoch:
    • Shuffle the dataset.
    • For each batch of size \( k \):
      • Calculate predicted values \( \hat{y} = wx + b \) for the batch.
      • Calculate gradients \( \nabla w, \nabla b \) averaged over the batch.
      • Update parameters: \( \theta = \theta - \eta \nabla \theta \).
Formulas
Batch Gradient: \( \nabla w_{batch} = -\frac{2}{k} \sum_{i \in Batch} (y_i - (wx_i + b))x_i \)
Batch Update: \( w = w - \eta \nabla w_{batch} \)
Simple Python Program
import numpy as np

X = np.array([1, 2, 3, 4], dtype=float)
Y = np.array([3, 5, 7, 9], dtype=float)
w, b = 0.0, 0.0
lr, batch_size = 0.1, 2

for epoch in range(100):
    indices = np.random.permutation(len(X))
    X_s, Y_s = X[indices], Y[indices]
    
    for i in range(0, len(X), batch_size):
        x_b = X_s[i:i+batch_size]
        y_b = Y_s[i:i+batch_size]
        
        y_pred = w * x_b + b
        dw = -2/len(x_b) * np.sum(x_b * (y_b - y_pred))
        db = -2/len(y_b) * np.sum(y_b - y_pred)
        
        w -= lr * dw
        b -= lr * db

print(f"Final w: {w:.2f}, b: {b:.2f}")
Expected Output

  b = 1.00
