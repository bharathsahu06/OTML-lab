Experiment 17: Quasi-Newton Method (BFGS)

1) Aim
Minimize the function: $$f(x_1,x_2)=x_1^2+2x_2^2$$ using Quasi-Newton method.
2) Description

Quasi-Newton Methods (BFGS) bridge the gap between gradient descent's simplicity and Newton's Method's speed. By building an approximation of the Hessian (or its inverse) from gradient information alone, Quasi-Newton methods achieve superlinear convergence without computing second derivatives explicitly. BFGS (Broyden–Fletcher–Goldfarb–Shanno) is the most popular variant because it maintains positive-definiteness and numerical stability.

The core innovation is the BFGS update that improves the Hessian approximation after each iteration using observed curvature changes (the difference between consecutive gradients). This is far cheaper than computing actual Hessians, yet provides enough second-order information to achieve fast convergence. Line search (like backtracking with Armijo condition) ensures global convergence even from poor starting points.

BFGS is practical and widely used: it requires only gradients (not Hessians), converges faster than gradient descent, and has better numerical properties than Newton's Method. Understanding BFGS reveals how to balance computational cost and convergence speed—a perennial trade-off in optimization.

3) Important Formulas

Objective: $$f(x)=x_1^2+2x_2^2, \quad x=\begin{bmatrix}x_1\\x_2\end{bmatrix}$$

Gradient: $$\nabla f(x)=\begin{bmatrix}2x_1\\4x_2\end{bmatrix}$$

Search direction: $$p_k=-H_k\nabla f(x_k)$$ where $H_k$ is inverse Hessian approximation.

Point update: $$x_{k+1}=x_k+\alpha_k p_k$$

Define: $$s_k=x_{k+1}-x_k, \qquad y_k=\nabla f(x_{k+1})-\nabla f(x_k)$$

BFGS inverse-Hessian update: $$H_{k+1}=(I-\rho_k s_k y_k^T)H_k(I-\rho_k y_k s_k^T)+\rho_k s_k s_k^T, \quad \rho_k=\frac{1}{y_k^Ts_k}$$

Stopping condition: $$\|\nabla f(x_k)\|_2<\varepsilon$$

4) Important Functions Used
  • f(x): objective value.
  • grad_f(x): gradient computation.
  • backtracking_line_search(...): finds step size satisfying Armijo condition.
  • bfgs_quasi_newton(...): performs full BFGS iterations.
5) Algorithm (Step by Step)
  1. Initialize point $x_0$, inverse-Hessian approximation $H_0=I$.
  2. Compute gradient $g_k=\nabla f(x_k)$.
  3. Compute descent direction $p_k=-H_k g_k$.
  4. Perform backtracking line search for $\alpha_k$.
  5. Update point $x_{k+1}=x_k+\alpha_k p_k$.
  6. Compute $s_k$ and $y_k$.
  7. Update inverse-Hessian approximation using BFGS formula.
  8. Repeat until gradient norm is below tolerance.
6) Mathematical Procedure (First Iteration)

Take initial point $x_0=[2,2]^T$ and $H_0=I$.

Gradient: $$g_0=\nabla f(x_0)=\begin{bmatrix}4\\8\end{bmatrix}$$
Direction: $$p_0=-H_0g_0=-\begin{bmatrix}4\\8\end{bmatrix}$$
With step $\alpha_0$ from line search: $x_1=x_0+\alpha_0p_0$.
Then compute: $s_0=x_1-x_0, \qquad y_0=g_1-g_0$ and update $H_1$ by BFGS formula.
7) Given Input
  • Function: $f(x_1,x_2)=x_1^2+2x_2^2$
  • Initial point: $x_0=[2,2]^T$
  • Tolerance: $10^{-8}$
8) Python Implementation & Output
