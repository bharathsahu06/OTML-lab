Experiment 13: Simple Distributed Gradient Descent Simulation

1) Aim
Simulate a simple distributed gradient descent algorithm for:
  • $x=[1,2,3,4,5,6]$
  • $y=[3,5,7,9,11,13]$
2) Description

Distributed Gradient Descent enables large-scale machine learning by partitioning data across multiple computers or nodes. Each node independently computes gradients on its local data, then a central aggregator (parameter server) combines them to update global model parameters. This approach is essential for modern machine learning: datasets too large for single machines, computational benefits from parallelization, and resilience through data distribution.

The elegance of distributed gradient descent lies in its simplicity and theoretical guarantees. Averaging local gradients from $K$ workers produces a gradient estimate with variance scaling like $1/K$—faster parallel learning. Communication happens only when aggregating gradients, making the method bandwidth-efficient. The method naturally handles data heterogeneity: as long as each worker computes correct local gradients, the globally aggregated gradient approximates the full-batch gradient.

This approach is the foundation of federated learning, parameter server architectures, and distributed deep learning frameworks like PyTorch Distributed and TensorFlow. Understanding this principle unlocks scalability: the same optimization algorithm works when computation is parallelized, enabling training on datasets and models impossible on single machines.

3) Important Formulas

Model: $$\hat{y}_i = wx_i + b$$

Global MSE: $$J(w,b)=\frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i-y_i)^2$$

Local gradient at worker $k$ (with $n_k$ samples): $$\frac{\partial J_k}{\partial w}=\frac{2}{n_k}\sum_{i\in D_k}(\hat{y}_i-y_i)x_i, \qquad \frac{\partial J_k}{\partial b}=\frac{2}{n_k}\sum_{i\in D_k}(\hat{y}_i-y_i)$$

Server aggregation (average over $K$ workers): $$\bar{g}_w=\frac{1}{K}\sum_{k=1}^{K} g_{w}^{(k)}, \qquad \bar{g}_b=\frac{1}{K}\sum_{k=1}^{K} g_{b}^{(k)}$$

Update rule: $$w\leftarrow w-\alpha\bar{g}_w, \qquad b\leftarrow b-\alpha\bar{g}_b$$

4) Important Functions Used
  • split_workers(...): splits data into worker partitions.
  • local_gradient(...): computes gradient on one worker.
  • distributed_gradient_descent(...): performs server-worker update loop.
  • mse_loss(...): computes full-data loss.
5) Algorithm (Step by Step)
  1. Initialize global parameters $w=0$, $b=0$.
  2. Split dataset into worker partitions (here, 2 workers).
  3. Send current $(w,b)$ to each worker.
  4. Each worker computes local gradients using its local data.
  5. Server averages local gradients.
  6. Server updates global $w$ and $b$.
  7. Compute global loss for monitoring.
  8. Repeat for fixed number of epochs.
  9. Return final parameters and predictions.
6) Mathematical Procedure (Complete Step-by-Step)

Given worker split:

  • Worker 1: $D_1=\{(1,3),(2,5),(3,7)\}$
  • Worker 2: $D_2=\{(4,9),(5,11),(6,13)\}$

Initial global parameters: $w_0=0$, $b_0=0$, $\alpha=0.01$.

1. Worker 1 at (w,b)=(0,0): Errors $[-3,-5,-7]$. $$g_w^{(1)}=\frac{2}{3}[(-3)(1)+(-5)(2)+(-7)(3)]=-22.6667$$ $$g_b^{(1)}=\frac{2}{3}[(-3)+(-5)+(-7)]=-10$$
2. Worker 2 at (w,b)=(0,0): Errors $[-9,-11,-13]$. $$g_w^{(2)}=\frac{2}{3}[(-9)(4)+(-11)(5)+(-13)(6)]=-112.6667$$ $$g_b^{(2)}=\frac{2}{3}[(-9)+(-11)+(-13)]=-22$$
3. Server aggregation (average): $$\bar{g}_w=\frac{g_w^{(1)}+g_w^{(2)}}{2}=-67.6667$$ $$\bar{g}_b=\frac{g_b^{(1)}+g_b^{(2)}}{2}=-16$$
4. Global update: $$w_1=0-0.01(-67.6667)=0.676667$$ $$b_1=0-0.01(-16)=0.16$$

5. Broadcast $(w_1,b_1)$ to workers and repeat same cycle for next epoch.

7) Given Input
  • $x=[1,2,3,4,5,6]$
  • $y=[3,5,7,9,11,13]$
  • Workers: $2$
  • Learning rate: $\alpha=0.01$
  • Epochs: $1000$
