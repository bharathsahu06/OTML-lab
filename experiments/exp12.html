Mini-batch gradient descent:

  • Divides data into small batches
  • Processes one batch at a time
  • Faster than batch GD, more stable than SGD
  • Better for parallel processing
ðŸ”§ Algorithm
Step 1: Initialize weights randomly
Step 2: Divide data into mini-batches
Step 3: For each batch, compute gradient
Step 4: Update weights
Step 5: Repeat for multiple epochs

import numpy as np # Mini-batch Gradient Descent for Linear Regression X = np.array([1, 2, 3, 4]).reshape(-1, 1) y = np.array([3, 5, 7, 9]) # Initialize w = 0.0 b = 0.0 lr = 0.01 batch_size = 2 epochs = 50 for epoch in range(epochs): # Shuffle data indices = np.random.permutation(len(X)) X_shuffled = X[indices] y_shuffled = y[indices] # Mini-batches for i in range(0, len(X), batch_size): X_batch = X_shuffled[i:i+batch_size] y_batch = y_shuffled[i:i+batch_size] # Predictions y_pred = w*X_batch + b # Gradients dw = -2/batch_size * np.sum((y_batch - y_pred) * X_batch) db = -2/batch_size * np.sum(y_batch - y_pred) # Update w -= lr * dw b -= lr * db # Loss y_pred_all = w*X + b loss = np.mean((y - y_pred_all)**2) if epoch % 10 == 0: print(f"Epoch {epoch}: Loss = {loss:.4f}, w = {w:.4f}, b = {b:.4f}") print(f"\nFinal: y = {w:.4f}x + {b:.4f}")
