Experiment 04: Stochastic Gradient Descent (SGD)

Home / Experiment 04
Aim

To implement Stochastic Gradient Descent (SGD) for linear regression, demonstrating how the model parameters are updated after processing each individual sample from the dataset.

Description

Unlike standard Gradient Descent which uses the entire dataset to compute gradients, Stochastic Gradient Descent (SGD) updates parameters for every single data point. This makes the optimization faster and able to handle huge datasets, though the path to the minimum is more "noisy" or "stochastic".

Algorithm
  1. Initialize parameters \( m \) and \( c \).
  2. For each epoch:
    • Shuffle the dataset (or select points randomly).
    • For each training example \( (x, y) \):
      • Calculate prediction \( \hat{y} = mx + c \).
      • Calculate Gradient for this single point: \( \frac{\partial L}{\partial m} = -(y - \hat{y})x \).
      • Update: \( m = m - \eta \cdot \frac{\partial L}{\partial m} \).
      • Update: \( c = c - \eta \cdot \frac{\partial L}{\partial c} \).
Formulas
Error per point: \( e = y - (mx + c) \)
Update Gradient: \( \nabla m = -e \cdot x, \quad \nabla c = -e \)
Parameter Update: \( \theta = \theta - \eta \nabla_\theta L \)
Mathematical Procedure

Assume point \( (2.5, 6.1) \) and model \( y = 1.0x + 0.5 \), \( \eta=0.01 \).

  • Prediction: \( 1.0(2.5) + 0.5 = 3.0 \)
  • Error: \( 6.1 - 3.0 = 3.1 \)
  • Update \( m \): \( 1.0 - 0.01(-3.1 \cdot 2.5) = 1.0 + 0.0775 = 1.0775 \)
  • Update \( c \): \( 0.5 - 0.01(-3.1) = 0.5 + 0.031 = 0.531 \)
Simple Python Program
import random

# Synthetic data: y = 2x + 1
data = [(x, 2*x + 1 + random.uniform(-1, 1)) for x in [random.uniform(0, 10) for _ in range(100)]]

m, c = 0.0, 0.0
eta = 0.01

for epoch in range(100):
    x, y = random.choice(data)
    y_pred = m * x + c
    error = y - y_pred
    
    # SGD updates
    m = m - eta * (-error * x)
    c = c - eta * (-error)

print(f"Final model: y = {m:.4f}x + {c:.4f}")
