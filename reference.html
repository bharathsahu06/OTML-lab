📚 Glossary & Reference

🔤 Optimization Terms
Convergence: When an algorithm's iterations produce values that stabilize and approach the optimal solution
Gradient (∇f): Vector of partial derivatives pointing in direction of steepest increase
Hessian: Matrix of second partial derivatives; describes curvature of function
Learning Rate (α): Step size parameter; controls how much we move in gradient direction
Local Minimum: Point where function value is lower than nearby points (may not be global minimum)
Global Minimum: Point with lowest possible function value
Objective Function: The function we want to minimize or maximize
Constraints: Restrictions on where feasible solutions can be (e.g., x ≥ 0)
Feasible Region: Set of all points satisfying all constraints
Regularization: Adding penalty terms to prevent overfitting and control model complexity
🧮 Mathematical Notation
∇f(x): Gradient of f at x; first derivative
∂f/∂x: Partial derivative of f with respect to x
||x||₁: L1 norm; sum of absolute values = |x₁| + |x₂| + ...
||x||₂: L2 norm (Euclidean); square root of sum of squares √(x₁² + x₂² + ...)
||x||²₂: L2 norm squared
Σ (summation): Sum over all elements
←: Assignment; replaces value on left with right-hand side
∈: Element of; belongs to a set
ℝⁿ: n-dimensional real numbers
🎯 Algorithm Types
Gradient Descent: Move in negative gradient direction with fixed step size
Stochastic GD (SGD): Use one random sample at a time; noisier but faster
Mini-batch GD: Balance between batch and stochastic; use small groups
Newton's Method: Use second-order (Hessian) information; faster convergence
Quasi-Newton (BFGS): Approximate Hessian; Newton-like speed without computing Hessian
Interior Point Method: Solve constrained problems with barrier functions
Conjugate Gradient: Exploit problem structure; efficient for large sparse systems
Sub-gradient Method: Handle non-smooth functions like L1 norm
🛠️ Regularization Techniques
L1 (Lasso): Penalty = λ||w||₁; creates sparse solutions (some weights exactly 0)
L2 (Ridge): Penalty = λ||w||²₂; shrinks weights smoothly, no exact zeros
Elastic Net: Combination of L1 and L2; λ₁||w||₁ + λ₂||w||²₂
Group Lasso: Apply L1 penalty to groups of weights, not individual weights
Early Stopping: Stop training when validation error starts increasing
🔗 Key Relationships
Gradient Descent Update:
x ← x - α∇f(x)
Move in negative gradient direction
Newton's Update:
x ← x - [∇²f(x)]⁻¹∇f(x)
Use curvature information (Hessian)
Loss with Regularization:
L(w) = DataLoss(w) + RegularizationTerm(w)
Balance accuracy and simplicity
Barrier Method:
min f(x) + (1/t)×penalty(x)
As t→∞, penalty→0, constraints enforced

⚙️ Hyperparameter Tuning
Learning Rate (α):
Too small: slow convergence. Too large: divergence/oscillation. Start with 0.01 or 0.001
Batch Size:
Larger: stable updates, slower. Smaller: noisy, faster. Try 32, 64, 128
Regularization Strength (λ):
Too small: overfitting. Too large: underfitting. Use cross-validation to tune
Number of Iterations/Epochs:
Check early stopping condition to avoid over-training
Initialization:
Random initialization usually fine. For Newton/BFGS, start close to optimum helps
✅ Convergence Criteria
Absolute Tolerance:
Stop when ||x_{k+1} - x_k|| < ε (small absolute change)
Relative Tolerance:
Stop when ||x_{k+1} - x_k|| / ||x_k|| < ε (small relative change)
Function Value Tolerance:
Stop when |f(x_{k+1}) - f(x_k)| < ε (objective barely changing)
Gradient Norm:
Stop when ||∇f(x)|| < ε (gradient near zero at optimum)
Maximum Iterations:
Always include fallback max iterations to prevent infinite loops
🐛 Debugging Tips
Loss Not Decreasing: Learning rate too high; reduce it
Very Slow Convergence: Learning rate too low; increase it
NaN Values: Numerical overflow; scale inputs or reduce learning rate
Check Gradients: Use finite differences to verify gradient computation
Print Intermediate Values: Debug by printing loss and parameters each iteration
Test Simple Cases: Start with known solutions before complex problems
📈 When to Use Each Algorithm
Gradient Descent: Baseline method; works almost everywhere; slow for large-scale
SGD/Mini-batch: Large datasets; online learning; parallel processing
Newton/BFGS: Small to medium problems; need fast convergence; smooth functions
Interior Point: Constrained problems; convex optimization problems
Conjugate Gradient: Large sparse linear systems; memory efficient
